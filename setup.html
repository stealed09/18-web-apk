🛠️ Developer Setup

One-time setup (3 minutes) — uses GitHub only, 100% FREE

✅ Method: GitHub Config File

config.json file is in the same GitHub repo.
Client edits it via GitHub web editor — no extra tools needed!

1

Files Structure

Your repo should have these files:

📂 my-app-releases/
├── 📄 index.html      ← Main website
├── 📄 guide.html      ← Client guide (Hindi)
├── 📄 setup.html      ← This file
├── 📄 config.json     ← APK link config ⭐
├── 📄 vercel.json     ← Vercel routing
├── 📄 README.md
└── 📄 .gitignore
2

config.json Structure

This is the config file that client will edit:

// config.json
{
  "apk_link": "https://github.com/.../app.apk",
  "version": "v1.0",
  "size": "24.5 MB",
  "updated": "Jan 2025"
}
3

Update index.html Config

Find these lines in index.html and set your GitHub details:

const GITHUB_USER = "stealed09";
const GITHUB_REPO = "my-app-releases";
const GITHUB_BRANCH = "main";

Also update the fallback APK link.

4

Update guide.html Links

In guide.html, update the edit links to point to YOUR repo:

<!-- Change this URL to your repo -->
href="https://github.com/stealed09/my-app-releases/edit/main/config.json"
5

Push to GitHub & Deploy

git add .
git commit -m "Setup complete"
git push origin main
  1. Go to vercel.com
  2. Login with GitHub
  3. "Add New Project" → Select repo
  4. Deploy → Done!
6

Give Client Access

  1. Add client as collaborator on GitHub repo
  2. Repo → Settings → Collaborators → Add → Enter their username
  3. Send them the guide page URL: yoursite.vercel.app/guide

⚠️ Client needs GitHub account

They need a free GitHub account to edit config.json. Help them create one if needed.
